# CareerLog

**CareerLog** 是一个基于命令行的个人求职信息管理系统。它专为求职阶段的候选人设计，用于高效地记录、存储和管理带有时间戳的申请记录、面试反馈及流程进度。

该项目采用纯 C++ 编写，不依赖第三方数据库引擎，通过层级化的数据结构（Database -> Group -> Record）帮助您清晰地梳理复杂的求职流程。

## 主要特性

**层级化数据管理**：采用 `Database` (数据库) -> `Group` (公司/项目组) -> `Record` (具体事件) 的三层架构，逻辑清晰。

**精确的时间追踪**：内置 `Date` 和 `TimePoint` 类，确保每一条记录都有准确的时间戳，支持跨时区标准化格式。

**上下文感知的命令行界面**：

- 提供类似 Shell 的交互式操作。
- 支持在“全局模式”和“特定组模式”之间切换，不同模式下可用指令动态变化。

**轻量级与高性能**：无外部依赖（如 SQL 驱动），启动迅速，内存占用极低。

**自动化测试套件**：包含完整的单元测试模块（覆盖日期处理、记录解析、群组管理等核心逻辑）。

## 快速开始

### 环境要求

编译器：支持 C++11 或更高版本 (如 GCC, Clang, MSVC)。

操作系统：目前仅支持Windows。

### 编译项目

在终端中进入项目根目录，使用以下命令编译：

代码

```
g++ src/*.cpp -I include -o careerlog -std=c++11
```

### 运行程序

编译成功后，直接运行可执行文件：

代码

```
./careerlog
```

您将看到 `[DB]>` 提示符，输入 `help` 查看可用命令即可开始使用。

## 使用指南

CareerLog 通过两级菜单来组织数据：

**Database 层级 (**`[DB]>`**)**：管理不同的公司或职位申请组。

**Group 层级 (**`[GRP:name]>`**)**：管理特定公司内的具体投递、笔试、面试记录。

### 常用工作流示例

假设您要记录对 "Google" 的申请流程：

1. **创建组**：在 `[DB]>` 模式下输入 `add_group Google`。
2. **进入组**：输入 `enter 0` (假设 Google 是第 1 个组)。
3. **添加记录**：
   - 提示符变为 `[GRP:Google]>`。
   - 输入 `add_record 2023-10-01 09:00:00 投递简历`。
   - 输入 `add_record 2023-10-05 14:30:00 电话面试`。
4. **查看进度**：输入 `print` 查看该组下的所有时间线。
5. **返回上级**：输入 `exit` 回到 `[DB]>` 模式。

### 指令速查表

| 模式      | 指令                              | 描述                     |
| --------- | --------------------------------- | ------------------------ |
| **通用**  | `help`                            | 显示当前可用的命令列表   |
| **通用**  | `print`                           | 打印当前层级的所有信息   |
| **[DB]**  | `add_group <name>`                | 创建一个新的公司/职位组  |
| **[DB]**  | `remove_group <id>`               | 删除指定 ID 的组         |
| **[DB]**  | `enter <id>`                      | 进入指定的组             |
| **[GRP]** | `add_record <date> <time> <info>` | 添加一条带时间戳的记录   |
| **[GRP]** | `remove_record <id>`              | 删除指定 ID 的记录       |
| **[GRP]** | `exit`                            | 退出当前组，返回 DB 层级 |

## 项目架构

项目遵循面向对象设计原则，核心类图如下：

`CommandProcessor`: 负责解析用户输入，维护当前的上下文状态（是在 DB 还是 Group 中），并分发命令。

`Database`: 顶层容器，管理所有的 `Group` 对象。

`Group`: 中间层容器，管理属于该公司的所有 `Record`。

`Record`: 最小数据单元，包含 `Date`, `TimePoint` 和文本信息。

## 运行测试

项目包含一个简易的测试框架（模拟 GTest 行为），用于验证核心逻辑的正确性。

在主函数中调用 `runAllTests()` 即可运行测试套件。测试覆盖了：

`test_Date_and_TimePoint`: 日期时间的合法性与格式化。

`test_Record`: 字符串解析与异常处理。

`test_Group`: 记录的增删与边界检查。

`test_Database`: 组的增删与查找。
